# Mahfazati Landing Page 💳

الموقع التعريفي (Landing Page) لمنظومة **محفظتي**، مصمم باستخدام التقنيات الأساسية لضمان أقصى درجات الأداء والبساطة.

## 📁 هيكلية المشروع (Project Structure)

تم تنظيم المشروع ليكون سهلاً وواضحاً للمطورين الجدد:

- **`main/`**: المجلد الرئيسي الذي يحتوي على ملف `index.html`.
- **`public/`**: يحتوي على الوسائط الثابتة مثل الصور (`images/`) والخطوط (`fonts/`).
- **`assets/`**: المصادر والملفات الخام المستخدمة في بناء التصميم.
- **`css/`**: ملفات التنسيق والأنماط.
- **`js/`**: ملفات البرمجة التفاعلية البسيطة.

```
Mahfazati-Web/
├── assets/           # الملفات المساعدة التي تعالج برمجياً
├── public/           # الملفات الثابتة التي يتم استدعاؤها مباشرة
│   ├── images/       # الصور، الأيقونات، واللوغو
│   └── fonts/        # الخطوط المستخدمة في الصفحة
├── css/              # ملفات التنسيق (Stylesheets)       # الملف الأساسي والوحيد للصفحة
├── index.html          # الصفحة الرئيسية (Landing Page)
├── about.html          # صفحة تعريفية إضافية
├── services.html       # صفحة شرح خدمات المحفظة
└── contact.html        # صفحة التواصل
```

## 🛠️ قواعد المساهمة (Contribution Rules) ⚠️

للحفاظ على استقرار المشروع، يمنع منعاً باتاً العمل المباشر على فرع الـ `main`. يجب اتباع الخطوات التالية عند الرغبة في إجراء أي تعديل:

1.  **إنشاء فرع جديد (Branch):** قم بإنشاء فرع خاص بالمهمة التي تعمل عليها أو الصفحة التي تعدلها.

    ```bash
    git checkout -b feature/your-task-name
    ```

2.  **العمل والتعديل:**
    قم بإجراء تعديلاتك داخل الفرع الجديد فقط.

3.  **طلب الدمج (Pull Request):**
    بعد الانتهاء من العمل ورفعه على GitHub، قم بفتح **Pull Request (PR)** لطلب دمج التعديلات مع الفرع الرئيسي.

4.  **المراجعة والدمج:**
    سيتم مراجعة الكود من قبل المسؤول عن المشروع قبل الموافقة على دمج التعديلات في فرع `main`.
